# CHANGELOG 範本

`CHANGELOG.md` 用於記錄專案的版本歷史與重大變更。以下提供範本供拷貝使用，建議採用語義化版本號（SemVer）管理。

```markdown
## vX.Y.Z - YYYY-MM-DD

### 新增
- [新增] 描述新增的功能、模板或設定。例如：新增世界觀規則 `gravity_rules.md`。

### 修正
- [修正] 描述修正的問題與影響。例如：修正角色 Ash 的能力上限描述。

### 改變
- [改變] 描述更動或移除的部分。例如：重構時間線，事件順序調整。

### 注意
- [注意] 如有破壞性變更，需提醒所有成員同步更新本地資料。
```

**使用方式**：

1. 新版本發布時，在檔案頂端插入一段新版本紀錄。
2. 按照新增、修正、改變三類分點敘述；不需要分類則可直接列出。
3. 若有重大更新需全員注意，可在「注意」區段列出。
