# TIMELINE 範本

時間線檔用於管理故事中重要事件的順序與關聯。請將每個事件按順序記錄於 `04_Timeline/` 中的 Markdown 檔案，確保不會出現時間或邏輯錯亂。以下是建議欄位：

```markdown
## 事件名稱

- **發生章節**：事件在小說中的位置，例如第 10 章。對於多章節事件，可列出範圍。
- **發生時間**：具體日期或年份，若世界觀有特殊曆法亦需註明。
- **發生地點**：事件發生的地理位置或區域。
- **參與角色**：所有重要角色名單。
- **事件經過**：概述事件發生的過程及重要細節。
- **造成結果**：事件對故事造成的直接後果，例如某國滅亡、角色重傷等。
- **後續影響**：事件長遠的影響，例如推動另一場戰爭或改變某角色的人生。
- **潛在衝突**：標註此事件可能與其他事件或設定產生的矛盾，方便日後檢查。
```

**注意**：

- 當角色的移動、康復或信息傳遞涉及時間跨度時，請合理計算所需時間，避免瞬移或過快恢復。
- 複雜事件可以分解為多個子事件，各自建立條目並於主事件中鏈結。
