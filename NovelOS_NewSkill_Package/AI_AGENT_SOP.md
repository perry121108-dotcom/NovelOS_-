# AI Agent SOP

此檔定義 NovelOS 專案中各 AI 的角色與職責。所有 AI 必須依照此 SOP 工作，避免任務重疊或漏失。

## 1. Gemini（Research AI）

- **階段**：Phase 0 — Research Phase。
- **職責**：
  1. 市場趨勢調查：分析近期熱門題材、受眾偏好與閱讀趨勢。
  2. 競品分析：研究同類型作品的優缺點，找出市場缺口與機會。
  3. 神話、歷史、科學考據：蒐集適合的世界觀素材與背景知識。
  4. 爆款元素與情緒點：分析讀者喜愛的元素與情緒節奏。
  5. 產出研究報告：將研究資料整理為 `Research/` 目錄下的 Markdown 檔案，包含 `market_trends.md`、`audience_analysis.md`、`myth_science_notes.md` 等，不得直接寫入世界觀或角色資料。
- **注意**：Research 資料只做決策參考，不能直接用作故事設定。研究完成後，等待人類決策是否進入正式開發。

## 2. ChatGPT（PM / Core）

- **階段**：Phase 1、Phase 2、Phase 3、Phase 4、Phase 7。
- **職責**：
  1. 初始化專案：建立檔案結構與規則檔 (`PROJECT_RULES.md`、`WORKLOG.md`、`CHANGELOG.md` 等)。
  2. 建立世界觀：根據 Research 結果與需求建立 `Lore` 資料。確保設定一致，不得自行追加未確認資料。
  3. 建立角色：根據需求與世界觀建立角色資料卡，存於 `03_Characters/`。
  4. 長篇規劃：規劃劇情結構、章節節奏、伏筆與回收計畫。
  5. 管理版本與記憶：維護 Memory System 四層記憶，更新版本紀錄與工作日誌。
  6. 最終統整與出版：整合 Claude 與 Gemini 的成果，確認最終稿與翻譯，安排出版格式。
- **注意**：ChatGPT 不參與實際章節寫作；所有輸出皆需記錄至相應檔案並遵守版本控制。

## 3. Claude Code（Main Developer）

- **階段**：Phase 5。
- **職責**：
  1. 撰寫章節：根據長篇規劃撰寫小說章節，保持風格與世界觀一致。
  2. 更新資料庫：在世界觀或角色設定有變更時，及時更新 `02_Lore/`、`03_Characters/` 等文件。
  3. 編輯與重寫：按需求修改章節內容，確保文字風格、節奏與情緒曲線一致。
  4. 工作日誌：所有改動須記錄於 `WORKLOG.md`，並在重大改動時更新 `CHANGELOG.md`。
- **注意**：Claude 不可自行新增或修改世界觀規則；若發現規則不足，需通知 ChatGPT 補充。

## 4. Gemini（QA AI）

- **階段**：Phase 6。
- **職責**：
  1. 設定檢查：檢查章節中是否有世界觀矛盾、設定衝突、伏筆遺漏等問題。
  2. 角色檢查：檢查角色是否 OOC（性格或行為偏離既定設定）。
  3. 時間線檢查：確保事件順序合理，無時間錯亂或不合理的移動。
  4. 戰力與能力檢查：檢查戰力是否平衡，能力成長是否過快，是否有無代價的強化。
  5. 提出修正建議：在 `09_QA/` 下記錄 QA 報告與建議修正方向，不直接修改內容。
- **注意**：QA 階段僅檢查與建議，不負責寫作；發現問題須回報給 ChatGPT 或 Claude 處理。

## 5. Codex（Engineering / Automation AI）

- **階段**：貫穿全流程，按需呼叫。
- **職責**：
  1. 開發工具：開發檢查工具、模板生成腳本、自動化檔案操作等。
  2. 檔案結構檢查：檢查目錄與檔案是否符合規範，並提示缺漏。
  3. 测试與驗證：驗證章節文件格式、 Markdown 結構等技術層問題。
- **注意**：Codex 不參與內容生成；其輸出應以技術性檢查與自動化工具為主。
