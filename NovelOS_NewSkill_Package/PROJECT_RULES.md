# PROJECT RULES

本檔列出 NovelOS 專案需遵守的核心規則與禁止事項。所有參與者（人類及 AI）皆須嚴格遵守，以確保長篇小說開發的品質與一致性。

## 一般原則

1. **資料分層**：Research 資料必須存放於 `Research/` 目錄，不得直接加入世界觀設定。正式設定存放於 `02_Lore/`、`03_Characters/` 等資料夾。
2. **流程順序**：必須先完成 Research Phase 並決策題材可行後，才可啟動正式開發。不能跳過研究直接創作。
3. **版本控制**：每次重大修改必須更新 `CHANGELOG.md`，並在 `WORKLOG.md` 記錄修改內容、影響範圍與版本號。
4. **工作日誌**：所有 AI 修改資料時必須立即更新 `WORKLOG.md`。未記錄之修改視為無效。

## 禁止事項

1. **禁止自行補完未確認設定**：未經審核的設定不可直接加入世界觀或角色。若需要新增，必須通過 ChatGPT 確認。
2. **禁止世界觀吃書**：不得與既定的世界規則或前文設定矛盾。任何調整需更新相關檔案並記錄版本號。
3. **禁止角色 OOC**：角色言行須符合其性格與成長線，不得無故改變性格或動機。
4. **禁止無代價升級**：能力成長必須有對應代價或限制，戰力不可無限膨脹。
5. **禁止不回收伏筆**：所有伏筆必須在合理章節回收並記錄於 `Foreshadow/`。不可遺忘伏筆或硬湊。
6. **禁止未更新版本紀錄**：任何重大設定變更若未更新 `CHANGELOG.md`，即視為無效。
7. **禁止未經研究直接開發**：若 Research Phase 未完成，不得進行世界觀或章節開發。

## 建議遵守

1. **溝通清晰**：各 AI 在修改或提出建議時，需使用清晰的文件和標籤，以便團隊理解。
2. **定期檢查**：每完成一定進度後，由 Gemini QA 檢查設定一致性並提出修正。
3. **重用模板**：創建資料時應優先使用範本文件，保持格式一致，便於搜尋與機器讀取。
4. **保護核心設定**：核心不可變設定僅允許 ChatGPT 修改；其他 AI 修改前需提出請求。

遵守本規則將能確保 NovelOS 專案的順利推進，並避免常見的長篇創作崩壞問題。
