# WORKLOG 範本

`WORKLOG.md` 用於記錄每次修改或新增內容的工作日誌。所有 AI 與創作者在修改任何檔案之前與之後，均須遵循本範本填寫日誌，以便團隊同步與版本追溯。

以下範本可複製到 `WORKLOG.md` 中，每次修改增加一段。

```markdown
## [YYYY-MM-DD] 工作項目標題（例：更新角色設定、修正世界觀漏洞）

- **負責人／AI**：ChatGPT / Claude / Gemini / Codex / 人類
- **版本號**：vX.Y.Z（如 v1.2.0）
- **修改內容**：
  - 簡要描述本次修改或新增的內容。
  - 指出涉及的檔案（例如：`02_Lore/world_rules.md`、`03_Characters/Ash.md`）。
- **影響範圍**：
  - 說明此修改可能影響的章節或設定，例如：涉及 04_Timeline 的某事件順序調整。
- **代辦事項**：
  - 如果因為此次修改需要後續工作或其他 AI 配合，請列出待辦。
- **風險／注意事項**：
  - 列出此修改可能帶來的風險或需特別注意的部分，便於 QA 關注。
```

**注意**：

- 每次修改請在 `WORKLOG.md` 最末端追加，不要覆蓋舊紀錄。
- 重大變更需同步更新 `CHANGELOG.md`。
- `版本號` 可遵循 [語義化版本 (SemVer)](https://semver.org/lang/zh-CN/)，依據修改性質增修。
